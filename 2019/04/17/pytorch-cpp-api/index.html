PyTorch C++ API初探

Posted on 2019-04-17 | In 深度学习 | Visitors:

在要求低延迟、脚本部署的生存环境，C++是更被青睐的语言。在之前版本的PyTorch提供的都是Python语言的接口，PyTorch1.0版本的发布带来了C++的前端，为生产环境的部署带来了方便。

备注，本文章基于的是LibTorch1.0，在cuda10 cudnn7环境下做的实验，最近PyTorch 1.3版本发布了，Torch Script的生成方式以及很多API都发生了较大变化，本文的一些例子也不再适用了。经过测试，本文提到的待解决的两个问题在新版本中也已经得到解决。 ——— 于 2019.10.13更新。

1. 将Pytorch Model转换为Torch Script

首先，C++能够理解的模型不是原生的Python版本的模型，而是需要通过Torch Script编译和序列化Python模型，C++调用序列化后的模型进行预测。

官方给出了两种将PyTorch模型转换为Torch Script的方法，第一种是通过torch.jit.trace方法，该方法缺点是在forward()中不能有复杂的条件控制，优点是操作比较方便；第二种是通过@torch.jit.script_method装饰器的方式，该方法需要重写PyTorch的模型，较为麻烦，但是允许forward()中有条件控制。

5. 待解决的问题

目前，如果模型中使用了LSTM等循环的网络结构，因为涉及init hidden state,所以如果想在c++中加载模型，且在GPU上预测会有问题，因为在调用to()方法将模型放置到显存中时，init hidden state并不会自动被放入显存。这个bug在python中也存在，但是可以使用重写父类的to()或者cuda()和cpu()方法解决，然而，在c++中并不可以。一种非常蠢的方法是在Python中用hard code指定好init hidden state的显卡位置，c++调用时也使用该显卡进行预测，这样就可以使用GPU计算了。

除了上面的问题以外，另一个问题是 如果在模型中使用了LSTM等循环网络的结构，且c++使用GPU进行预测，则代码退出时会报core dump。这个问题怀疑是返回的智能指针在释放对应内存空间时，没有处理好显存的释放，可能和init hidden state也有关系，目前我已经开了issue，等待官方的fix。
